Baseado na sua solicitação de funcionalidade completa de **HIT (High Intensity Training)** para cardio, projetei a solução para se integrar perfeitamente ao editor existente.

**Análise Técnica:**
O sistema atual já suporta configurações avançadas via JSON (`advanced_config`). Não precisamos alterar o banco de dados, apenas a interface do editor (`ExerciseEditor.js`) para ler e gravar esses dados.

**Plano de Implementação (HIT Cardio):**

1.  **Modificar `ExerciseEditor.js`:**
    *   **Toggle de Ativação:** Adicionar um switch "Modo HIT" dentro da seção de Cardio.
    *   **Campos Condicionais:** Quando ativado, exibir campos específicos:
        *   ⏱️ **Tempo de Trabalho** (ex: 30s)
        *   ⏸️ **Tempo de Descanso** (ex: 10s)
        *   🔄 **Rounds/Séries** (ex: 10)
        *   🔥 **Intensidade** (Baixa/Média/Alta)
    *   **Validação:** Impedir que o tempo de descanso seja maior que o de trabalho (com aviso visual).
    *   **Resumo:** Exibir um "badge" ou resumo textual (ex: *"HIT: 10 rounds de 30s/10s"*) para fácil visualização.

2.  **Estrutura de Dados (JSON):**
    *   Os dados serão salvos em `workout_items.advanced_config` (ou no `setDetails[0].advanced_config` como descobri na pesquisa), garantindo compatibilidade com o resto do app.
    *   Formato: `{ isHIT: true, workSec: 30, restSec: 10, rounds: 10, intensity: 'high' }`.

3.  **Visualização no Editor:**
    *   Garantir que, ao carregar um treino existente, o editor reconheça se é HIT e preencha os campos automaticamente.
