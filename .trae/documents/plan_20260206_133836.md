Baseado na análise do projeto, identifiquei exatamente o motivo do deploy não estar "passando" pelo GitHub (Actions):

### 1. Ausência de Arquivos de Configuração do GitHub
O diretório `.github/workflows`, onde ficam as automações do GitHub Actions (como pipelines de CI/CD), **não existe** neste projeto. Sem esses arquivos, o GitHub não executa nenhuma tarefa visual na aba "Actions" quando você faz um push.

### 2. Integração via Vercel (Oculta)
Pelo arquivo `DEPLOY.md`, o projeto utiliza a **Integração Nativa da Vercel**.
- **Como funciona:** A Vercel "escuta" silenciosamente seu GitHub. Quando você faz push na `main`, a Vercel puxa o código e faz o deploy lá nos servidores dela.
- **O que parece:** Como não há uma "Action" rodando no GitHub, parece que "não passou pelo GitHub", mas na verdade passou (o código foi armazenado lá) e a Vercel agiu em segundo plano.

### 3. O Script `npm run deploy` Burlador
No seu `package.json`, o comando de deploy está configurado assim:
`"deploy": "git config user.email ... && npx vercel --prod"`

**Este comando é perigoso** e explica sua dúvida:
- Quando você roda `npm run deploy` no seu computador, ele **ignora o GitHub completamente**.
- Ele pega os arquivos da sua pasta local (mesmo sem commit) e manda direto para a Vercel.
- O `DEPLOY.md` inclusive alerta para **não usar** esse comando em produção por este motivo.

### 🚀 Plano de Ação (Sugerido)
Para corrigir isso e ter um fluxo profissional onde tudo passa visivelmente pelo GitHub:

1.  **Criar Workflow de CI/CD:** Posso criar um arquivo `.github/workflows/deploy.yml`. Isso fará com que, ao dar push, o GitHub mostre uma "bolinha amarela/verde" indicando o progresso do build antes de enviar para a Vercel.
2.  **Ajustar Script:** Alterar o `npm run deploy` para apenas avisar "Por favor, faça push para a main para deployar", evitando erros manuais.
