# Implementar Controle de Acesso e Aprovação de Contas

Para garantir o controle de acesso, vou implementar um sistema onde novos usuários só podem entrar após aprovação de um admin.

## 1. Criar API de Aprovação (`/api/admin/access-requests/action`)
- **Objetivo:** Permitir que o admin aprove ou rejeite solicitações pendentes.
- **Funcionamento:**
    - Se **Aprovado**:
        - Cria o usuário no Supabase Auth usando `admin.auth.admin.createUser` (com senha temporária ou link de convite).
        - Envia e-mail de boas-vindas com link de acesso.
        - Atualiza status para `approved`.
    - Se **Rejeitado**:
        - Atualiza status para `rejected`.
        - Envia e-mail notificando (opcional).

## 2. Reforçar Middleware (RBAC)
- Como você ainda não autorizou mexer no `middleware.ts` (Auth Lock), vou focar na API de aprovação que é o core do controle.
- Se desejar aplicar o bloqueio de rotas agora, responda com **"OVERRIDE AUTH LOCK"** após eu implementar a API.

## 3. Fluxo Final
1. Usuário preenche "Pedir Acesso" na tela de login.
2. Admin vê a solicitação no painel e clica em "Aprovar".
3. Sistema cria a conta e manda e-mail.
4. Usuário acessa e define sua senha definitiva.

Vou criar a rota de API que falta para o botão de aprovação funcionar.
