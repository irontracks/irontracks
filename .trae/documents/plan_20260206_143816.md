Entendi a necessidade de uma solução universal e robusta. O problema do "vídeo preto" é clássico em exportações via Canvas quando o navegador bloqueia o acesso aos frames do vídeo por questões de segurança (CORS) ou quando a decodificação do vídeo não acompanha a taxa de renderização.

Vou implementar um modelo universal de processamento de vídeo baseado em **WebCodecs API** (onde suportado) com fallback para renderização síncrona via Canvas, garantindo que nenhum frame seja perdido ou fique preto.

### Solução Universal de Processamento de Vídeo

1.  **Núcleo de Renderização Universal (Engine):**
    *   Criarei um novo módulo `VideoCompositor` que abstrai a complexidade do navegador.
    *   **Correção do "Vídeo Preto":** O sistema irá pré-carregar o vídeo e usar `createImageBitmap` ou `requestVideoFrameCallback` para extrair frames *somente* quando eles estiverem decodificados e prontos. Se o frame não estiver pronto, o gravador *espera*, garantindo sincronia perfeita e zero frames pretos.
    *   **Padronização 1080x1920 @ 30fps:** O Canvas de saída será forçado para estas dimensões. Qualquer vídeo de entrada será redimensionado (cover/contain) automaticamente para preencher este espaço sem distorção.

2.  **Tratamento de Áudio Robusto:**
    *   Usarei `AudioContext` para extrair o áudio do vídeo original, processá-lo (normalização opcional) e mixá-lo com o vídeo final. Isso resolve o problema de vídeos mudos ou dessincronizados.

3.  **Compatibilidade de Codecs (Transcodificação Automática):**
    *   **iOS/Mac:** Forçar codec `video/mp4` (H.264 + AAC) para compatibilidade nativa com galeria e Instagram.
    *   **Android/Windows/Linux:** Preferir `video/mp4` (H.264), com fallback para `video/webm` (VP9) se H.264 não estiver disponível via hardware.
    *   **Validação de Entrada:** O sistema aceitará qualquer formato que o navegador consiga decodificar.

4.  **Compressão Otimizada:**
    *   Configurarei o bitrate dinamicamente: 5 Mbps para 1080p (qualidade visual alta, tamanho aceitável) em vez de deixar o navegador decidir (que muitas vezes joga para 20Mbps ou 1Mbps).

### Implementação

Vou substituir a lógica atual do `StoryComposer.tsx` por este novo motor robusto.

**Arquivos:**
*   `src/components/StoryComposer.tsx`: Integração completa com o novo motor.
*   `src/lib/video/VideoCompositor.ts` (Novo): O cérebro do processamento universal.
