Vou implementar um sistema completo de **Gestão e Visualização de Créditos VIP**, abrangendo tanto o relatório administrativo quanto a experiência do usuário final.

### 1. Painel Administrativo (Relatórios)
*   **Backend (RPC `admin_get_vip_stats`)**:
    *   Criarei uma função no banco de dados para agregar estatísticas de uso em tempo real.
    *   **Métricas**: Total de usuários por plano (Free, Start, Pro, Elite), Capacidade Total de Créditos vs. Uso Real.
*   **Frontend (AdminPanelV2)**:
    *   Nova aba **"Relatórios VIP"**.
    *   Tabela detalhada mostrando a taxa de utilização de cada ferramenta (Chat, Insights, Wizard) por nível de assinatura.
    *   Gráficos de barra para visualizar o consumo global.

### 2. Interface do Usuário (Meus Créditos)
*   **Nova API (`/api/user/vip-credits`)**:
    *   Endpoint dedicado para retornar o saldo atual do usuário (ex: `chat: { used: 5, limit: 10 }`).
*   **Atualização das Ferramentas**:
    *   **Coach Chat**: Exibir contador "Mensagens hoje: 5/10" no topo do chat.
    *   **Workout Wizard**: Mostrar "Gerações restantes nesta semana: 1/3" no modal de criação.
    *   **Insights**: Adicionar indicador "Análises disponíveis: 2/3" na tela de relatório.
    *   **Feedback Visual**: Se o usuário atingir 80% do limite, o contador ficará amarelo; 100%, vermelho com link para upgrade.

### Passos da Implementação
1.  Criar a função RPC no banco de dados para o relatório admin.
2.  Criar o endpoint de API para consulta de saldo do usuário.
3.  Implementar a aba de relatórios no `AdminPanelV2`.
4.  Atualizar os componentes `CoachChatModal`, `WorkoutReport` e `WorkoutWizardModal` com os contadores de crédito.
