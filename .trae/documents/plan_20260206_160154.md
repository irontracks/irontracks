Vou refinar a interface dos botões para evitar confusão visual e sobreposição de textos.

### Diagnóstico
Atualmente, quando o app está processando (gerando o vídeo), **ambos** os botões mudam o texto para "PROCESSANDO...", o que pode causar poluição visual ou a sensação de que o texto está "encavalado" se o layout quebrar. Além disso, não dá para saber qual ação está rodando.

### Solução UX
Vou implementar um estado de "Ação em Progresso" para que apenas o botão que você clicou mostre o status de carregamento, enquanto o outro apenas fica desabilitado.

1.  **Estado `busyAction`:** Diferenciar se estamos `'postando'` ou `'baixando'`.
2.  **Botão POSTAR:**
    *   Se estiver postando: Mostra ícone de **Loading** (girando) + "PROCESSANDO..."
    *   Se estiver baixando: Fica desabilitado, mantendo o texto "POSTAR NO IRONTRACKS".
3.  **Botão BAIXAR:**
    *   Se estiver baixando: Mostra ícone de **Loading** + "PROCESSANDO..."
    *   Se estiver postando: Fica desabilitado, mantendo o texto "BAIXAR / COMPARTILHAR".

Isso elimina qualquer chance de texto sobreposto e deixa a interface mais limpa.
