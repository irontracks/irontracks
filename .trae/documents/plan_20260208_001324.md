# Implementar Controle de Acesso e Aprovação

Vou implementar o sistema de aprovação de usuários e proteger as rotas administrativas.

## 1. API de Aprovação (`/api/admin/access-requests/action`)
Criarei o endpoint que processa as solicitações:
- **Aprovar:** Cria o usuário no Supabase Auth, envia convite por e-mail e marca a solicitação como aprovada.
- **Rejeitar:** Apenas marca a solicitação como rejeitada.

## 2. Middleware de Segurança (RBAC)
Atualizarei o `middleware.ts` para proteger rotas sensíveis:
- **Rotas `/admin`:** Apenas usuários com role 'admin'.
- **Rotas `/teacher`:** Apenas usuários com role 'teacher' ou 'admin'.
- **Bloqueio:** Usuários sem permissão serão redirecionados para o `/dashboard` (ou `/auth/login` se não autenticados).

Isso garante que apenas pessoas autorizadas acessem o painel administrativo e que novos usuários precisem de aprovação explícita para entrar.
