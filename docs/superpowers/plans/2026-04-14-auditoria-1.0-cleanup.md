# Auditoria 1.0 — Cleanup & Correções Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminar os 507 ESLint warnings que bloqueiam o deploy, apagar 86 arquivos duplicados gerados por worktrees, corrigir `<img>` sem otimização e adicionar rate limiting nas rotas VIP/LLM críticas.

**Architecture:** Quatro grupos de trabalho independentes: (A) deleção de arquivos duplicados — zero risco, (B) correção de unused vars / imports — mecânico, arquivo a arquivo, (C) correções de acessibilidade a11y em lote — padrão repetitivo `<input>` + `<div onClick>`, (D) rate limiting em rotas VIP. Os grupos A e B podem ser executados em paralelo; C depende de B estar completo para evitar conflito de edição nos mesmos arquivos; D é completamente independente.

**Tech Stack:** Next.js 16, TypeScript 5.9 strict, ESLint 9 + jsx-a11y, Tailwind CSS v4

```bash

```


- `<div onClick>` para itens de lista clicáveis

**Padrões de correção:**

**Padrão 1 — Input com label visual em `<div>`:**
```tsx
// ANTES (viola label-has-associated-control e control-has-associated-label):
<div className="space-y-1">
  <div className="text-xs text-neutral-400">Peso (kg)</div>
  <input inputMode="decimal" value={...} onChange={...} placeholder="Ex: 80" className="..." />
</div>

// DEPOIS:
<div className="space-y-1">
  <label htmlFor="field-peso" className="text-xs text-neutral-400">Peso (kg)</label>
  <input id="field-peso" inputMode="decimal" value={...} onChange={...} placeholder="Ex: 80" className="..." aria-label="Peso em kg" />
</div>

// OU (mais simples quando não se quer mudar o elemento label):
<div className="space-y-1">
  <div className="text-xs text-neutral-400" id="label-peso">Peso (kg)</div>
  <input inputMode="decimal" value={...} onChange={...} placeholder="Ex: 80" className="..." aria-labelledby="label-peso" />
</div>
```

**Padrão 2 — Backdrop de modal `<div onClick>` (click-events-have-key-events + no-static-element-interactions):**
```tsx
// ANTES:
<div className="fixed inset-0 ..." onClick={() => setModal(null)}>
  <div className="..." onClick={(e) => e.stopPropagation()}>

// DEPOIS (adicionar role e keyboard handler ao backdrop):
<div
  className="fixed inset-0 ..."
  role="button"
  tabIndex={-1}
  aria-label="Fechar modal"
  onClick={() => setModal(null)}
  onKeyDown={(e) => { if (e.key === 'Escape') setModal(null) }}
>
  <div className="..." onClick={(e) => e.stopPropagation()}>
```

**Padrão 3 — Item de lista clicável `<div onClick>`:**
```tsx
// ANTES:
<div onClick={() => handleSelect(item)} className="cursor-pointer ...">

// DEPOIS:
<button
  type="button"
  onClick={() => handleSelect(item)}
  className="cursor-pointer w-full text-left ..."
>
```
